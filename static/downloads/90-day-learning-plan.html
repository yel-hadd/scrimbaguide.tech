Your First 90 Days Learning Plan

A week-by-week learning checklist anchored to Scrimba's Frontend Developer Path (81.6 hours). Adapt for your pace — 10h/week takes 8–9 weeks, 20h/week takes 4–5 weeks.

Month 1 — Foundation (Weeks 1–4)

Phase 1: Learn the syntax — don't build yet Scrimba path: Weeks 1–4
Week 1 HTML + CSS basics. Scrimba: Web Dev Basics. Goal: understand the box model, flexbox, basic layouts. Don't memorize — understand the pattern.
Week 2 JavaScript fundamentals. Scrimba: Learn JavaScript. Variables, functions, arrays, objects. Do every challenge — don't skip.
Week 3 JavaScript continued. Scrimba: DOM manipulation, events, fetch API. Build the in-lesson projects without looking at the solution first.
Week 4 First mini-project (solo). Build a simple to-do list from scratch WITHOUT a tutorial. Use MDN for reference only. This is your first proof that the knowledge stuck.
Month 1 Checkpoint: Can you build a working to-do list from a blank HTML file without referring to any tutorial? If yes, you're on track. If no, repeat Week 4 with a different project (e.g., a simple quiz or calculator).

Month 2 — React and Real Projects (Weeks 5–8)

Phase 2: Learn React — build things that look like real apps Scrimba path: Weeks 5–8
Week 5 React basics. Scrimba: Learn React. Components, props, state, hooks (useState, useEffect). Watch every scrim and type every exercise.
Week 6 React continued + APIs. Fetch data from a public API (e.g., OpenWeather, PokeAPI). Build a 3-component app that displays real data.
Week 7 Portfolio project #1. Take a Scrimba lesson project and add 3 new features. Push to GitHub. Write a README. This is now your first portfolio piece.
Week 8 TypeScript introduction. Scrimba: TypeScript courses. Add types to your React project. TypeScript is now required for most frontend job listings.
Month 2 Checkpoint: You should have 1 GitHub repo with a working React + API project and a README. Show it to someone who doesn't code — can they use it without explanation?

Month 3 — Portfolio and Job Prep (Weeks 9–13)

Phase 3: Build portfolio. Start applying. Scrimba path: Weeks 9+
Week 9 Portfolio project #2. Build something from scratch using an external API you choose. No tutorial — just MDN and the API docs.
Week 10 Portfolio project #3 + LinkedIn. Build a tool that solves a real problem (for you or a friend). Set up your LinkedIn developer profile using the Scrimba Getting Hired module.
Week 11 Interview prep begins. Scrimba: JavaScript Interview Challenges + React Interview Questions. 1 coding challenge per day.
Week 12–13 Apply + learn in public. 5–10 high-quality applications per week. Post weekly progress on LinkedIn. Ask for code reviews in the Scrimba Discord.
Month 3 Checkpoint: 3 portfolio projects on GitHub + LinkedIn profile updated + 20+ applications sent. If you've done this, you're statistically ahead of most junior applicants.

Daily Habit Stack (Any Phase)

Time Block Activity Why
15 min — morning Review yesterday's code without looking at solutions Active recall beats re-reading
60–90 min — main session Scrimba course / project building Deep work, no multitasking
15 min — end of session Write 3 things you learned today Forces recall, builds portfolio content
30 min (optional) One LeetCode Easy / coding challenge Interview prep starts early
