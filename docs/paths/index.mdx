# Which Scrimba career path should you choose?

**Quick answer:** Scrimba has **four** structured career paths—**Frontend** (~82h), **Fullstack** (~108h), **Backend** (~30h), and **AI Engineer** (~11h). All paths require **Scrimba Pro**. Beginners who want a job should start with **Frontend**; people who want the broadest hireable stack choose **Fullstack**; people who already know frontend basics and want APIs/servers/SQL choose **Backend**; working developers who want agents, RAG, and AI app patterns choose **AI Engineer**—usually **after** you can ship a simple web app.

*Last reviewed: March 2026. Path durations and catalog change on Scrimba’s side; verify module lists on each path page.*

## Who each path is for (2026)

### Career switchers targeting a first developer role

Most people need a **credible portfolio** and **interview-ready fundamentals**. Frontend is the most common starting line because hiring pipelines still screen heavily for JavaScript + React + CSS. If you can commit **high weekly hours** and want the widest project surface area, Fullstack is the “one track” option—just expect a longer calendar commitment.

- Start here: [Frontend Developer path](/docs/paths/frontend-developer-path) or [Fullstack Developer path](/docs/paths/fullstack-developer-path)
- Pair with: [Career change to coding (2026)](/blog/career-change-to-coding-2026), [6-month study plan](/docs/paths/study-plan)

### Students (CS or not)

**CS students** often need practical web skills on top of theory—React, TypeScript, APIs, and shipped projects. See [Scrimba for CS students](/docs/paths/scrimba-for-cs-students). Non-CS students usually mirror career switchers: Frontend first unless you already have proof you can build UIs.

### Working professionals learning on the side

Low weekly hours usually favor **one** focused path and a steady cadence—not hopping between four catalogs. Frontend or Backend (if you already know JS) tends to fit constrained schedules better than Fullstack, which is a marathon.

- Pair with: [Learn to code with a full-time job](/blog/learn-to-code-full-time-job)

### Developers adding backend or AI

**Backend path** is for people who are already comfortable building pages or components and want servers, databases, and deployment depth. **AI Engineer** is for people who can ship apps and want applied AI patterns—if you are still shaky on JavaScript, do Frontend modules first so you are not debugging syntax and LLM prompts at the same time.
