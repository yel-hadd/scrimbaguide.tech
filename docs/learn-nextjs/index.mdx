# Next.js Learning Roadmap

This hub mirrors the [official Next.js docs](https://nextjs.org/docs) to **Scrimba’s free App Router course**, setup, routing, Server vs Client Components, data fetching, loading UI, and Server Actions, so each concept you read has a **typed-in-the-browser** counterpart.

## Who this is for

React devs who can build SPAs but freeze when `app/`, Server Components, and `loading.tsx` show up, and anyone on the [Fullstack path](/docs/paths/fullstack-developer-path) who wants Next to feel concrete.

## 2. Route and navigate like Next expects

**Goal:** Map folders to URLs, share layouts, and add dynamic segments.

Next.js uses a **file-system based router**. Folders define routes.

- **Linking:** `Link` component for client-side navigation.
    - [Scrimba Lesson: The Link Component](https://scrimba.com/learn/nextjs/link-component-c08?via=u42d4986)
- **Dynamic Routes:** `[slug]/page.js` for dynamic content.
    - [Scrimba Lesson: Dynamic Routes](https://scrimba.com/learn/nextjs/dynamic-routes-c010?via=u42d4986)

## 3. Pick Server vs Client Components on purpose

**Goal:** Stop shipping giant JS bundles when HTML would do.

| Type | When to use | Scrimba Lesson |
|---|---|---|
| **Server Components** | Data fetching, database calls, sensitive logic. (Default) | [Server Components](https://scrimba.com/learn/nextjs/server-components-c012?via=u42d4986) |
| **Client Components** | Interactivity, `onClick`, `useState`, `useEffect`. | [Client Components](https://scrimba.com/learn/nextjs/client-components-c014?via=u42d4986) |

## 4. Fetch and stream data without spinners everywhere

**Goal:** Let Server Components own reads; use Suspense + `loading.tsx` for UX.

In the App Router, you can fetch data directly in your Server Components using `async/await`.

- **Fetching Data:** `const data = await fetch(...)`
    - [Scrimba Lesson: Fetching Data](https://scrimba.com/learn/nextjs/fetching-data-c016?via=u42d4986)
- **Loading UI:** `loading.js` for instant loading states.
    - [Scrimba Lesson: Loading UI](https://scrimba.com/learn/nextjs/loading-ui-c018?via=u42d4986)

## 5. Mutate with Server Actions (skip boilerplate APIs)

**Goal:** Post forms straight to server functions when it fits your app.

Instead of writing a separate API endpoint (`/api/submit`), you write a function that runs on the server and call it directly from your form.

-> **This is a key part of the [Fullstack Career Path](https://scrimba.com/fullstack-path-c0fullstack?via=u42d4986).**
