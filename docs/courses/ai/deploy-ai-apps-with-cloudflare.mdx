## The problem: great model, fragile deployment

Localhost demos do not answer what interviewers and users actually ask: **What happens under load? Across regions? When someone abuses the endpoint?** If you cannot explain hosting, latency, and abuse resistance, the feature never leaves the hackathon.

## How this course helps

You focus on **Cloudflare-oriented patterns** for running AI workloads closer to users, with security and availability in the same mental model as your API code. Scrimba’s **interactive format** keeps you editing real requests and config in the lesson, closer to how you will iterate at work than a passive video.

This sits naturally on the [AI Engineer Path](/docs/paths/ai-engineer-path) **after** you are comfortable calling models from JavaScript.

**Honest scope:** Pricing tiers, org compliance, and vendor limits change often, use Scrimba for the **mental model**, then confirm details in Cloudflare’s current docs for production.

## Prerequisites

Solid **JavaScript** and basic **API** experience. If LLM calls are still new, take [Intro to AI Engineering](/docs/courses/ai/ai-engineering) first.

## Who this is for

- **You:** You ship or want to ship **AI features** and need **edge deployment** vocabulary fast.
- **Skip for now:** You have **no programming background**, or you only want **desktop Python** notebooks.
