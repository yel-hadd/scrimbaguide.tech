## The Kevin Powell question (read this first)

If you have spent any time learning CSS on YouTube, you already know Kevin Powell. His channel is one of the largest CSS resources on the web, with over a million subscribers and roughly a thousand free videos covering Flexbox, Grid, container queries, `:has()`, animations, the lot. The honest case for paying for Scrimba Pro to get Kevin's [Responsive Web Design bootcamp](/docs/courses/css/responsive-web-design) (15.1 hours) over his free YouTube channel is straightforward:

- **YouTube** is a deep, free reference. Search for "Kevin Powell flexbox align-items" and you will find the answer. Great for fixing a specific bug, less great for sequencing your learning from zero.
- **Scrimba** gives you the same instructor in an **ordered, interactive bootcamp** where you build four real layouts (a blog, a landing page, a banner, a company site) and you edit the CSS inside the lesson. That structure is the value.

Plenty of learners stay on YouTube and never pay. That is a valid choice, and we will say so. If you have already watched 40 of his videos and you still cannot ship a responsive layout end to end without copying, the bootcamp is what closes the gap. If you have not yet watched him at all, start free on YouTube. There is no shame in that order.

> "I love that I get to teach in a format that's actually interactive. You're not just watching me code, you're coding along with me." Kevin Powell, on teaching CSS at Scrimba ([Scrimba Podcast](https://podcast.scrimba.com/62/transcript)).

## You are on this page if

You need **hireable CSS** (Flexbox, Grid, responsive design, Tailwind, animations) and you want one ordered place to find every Scrimba course with duration, level, and Free vs Pro called out honestly. You learn by typing in the lesson, not by binge-watching a playlist.

## Recommended order, beginner to advanced

1. **HTML and CSS foundations.** Start free with Kevin Powell's HTML & CSS Crash Course (4 hrs, free) or Per Borgen's longer [Learn HTML and CSS](/docs/courses/css/html-and-css) (5.7 hrs, free). Pick one; do not do both.
2. **Layout pillars.** [Flexbox](/docs/courses/css/flexbox) (52 min) then [CSS Grid](/docs/courses/css/css-grid) (63 min, free). These two carry every modern layout.
3. **Responsive design.** Kevin Powell's [Responsive Web Design bootcamp](/docs/courses/css/responsive-web-design) (15.1 hrs, Pro). This is the deepest single course in the catalog and the main reason to pay for Pro here.
4. **A utility framework.** [Tailwind CSS](/docs/courses/css/tailwind-css) (2.2 hrs, Pro), then the free [Tailwind product card build](/docs/courses/css/a-product-card-with-tailwind-css) (75 min). Tailwind is the default for new React and Next.js projects.
5. **Polish.** [CSS Animations](/docs/courses/css/css-animations) (2 hrs), [CSS Variables](/docs/courses/css/css-variables) (29 min), [CSS Challenges](/docs/courses/css/css-challenges) (2.6 hrs, free, useful as reps).
6. **Design depth (optional).** Gary Simon's [UI Design Fundamentals](/docs/courses/css/ui-design-fundamentals) (70 min) then his full [Learn UI Design](/docs/courses/css/ui-design) (8.6 hrs, Pro) if you want the designer's eye, not just the syntax.

## Modern CSS features the courses cover

The catalog stays current with the features teams actually ship in 2026:

- **Container queries** for component-scoped responsive design.
- **`:has()` selector** (parent selectors) for form validation states and adjacent styling.
- **Aspect ratio** for cards and embeds without padding hacks.
- **CSS custom properties** for theming and design tokens.
- **Cascade layers** (`@layer`) for predictable specificity in large codebases.

Kevin's responsive bootcamp covers these in production-shaped projects, not as isolated demos.

## Honest gaps

We will not pretend this catalog covers everything:

- **No SCSS or Sass deep dive.** No SCSS deep dive on Scrimba. Bridge: Kevin Powell's own [YouTube channel](https://www.youtube.com/@KevinPowell) remains the standard reference for CSS and preprocessors, and the [Sass documentation](https://sass-lang.com/documentation/) covers conventions directly.
- **No CSS-in-JS specifics.** No styled-components, no Emotion, no vanilla-extract. Bridge: Josh Comeau's [styled-components guide](https://www.joshwcomeau.com/css/styled-components/) for the canonical write-up.
- **No design tool coverage.** No Figma, no Adobe XD. Gary Simon teaches UI design **using HTML and CSS**, which is unusual; it suits developer-designers, not pure visual designers. Bridge: the [Figma Learn channel](https://www.youtube.com/@Figma) for the tool itself.
- **Bootstrap and Bulma are short tasters.** Treat them as 60-minute orientation, not mastery.
- **Accessibility (WCAG, ARIA) is covered in passing.** Solid, but not a dedicated accessibility track. Bridge: the [A11y Project checklist](https://www.a11yproject.com/checklist/) and Sara Soueidan's [accessible components series](https://www.sarasoueidan.com/blog/).

For everything inside the modern-CSS-plus-Tailwind-plus-UI-design lane, the catalog is strong. For a third-party take on Kevin's bootcamp, see his own [dev.to announcement](https://dev.to/kevinpowell/learn-to-master-css-with-the-responsive-web-design-bootcamp-2i4o) and community discussion under the [dev.to #css tag](https://dev.to/t/css).

## Why scrims work for CSS specifically

CSS bugs are visual. You change `align-items` and the box moves; you change `flex-basis` and it does not. Watching someone else's video means you guess at why it broke. In a scrim you **pause and change the value yourself**, in the same browser, and see the box move. That feedback loop is the entire point of the format and it matters more for CSS than for almost any other topic.
