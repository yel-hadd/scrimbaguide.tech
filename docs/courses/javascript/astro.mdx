## About This Course

Learn Astro, the modern web framework for building fast, content-rich websites using components and Markdown. In this course, you’ll learn how to create and deploy a personal website from scratch, combining performance with a fun and flexible developer experience.

This Intermediate-level course covers 2.1 hrs of content. A [Scrimba Pro](/docs/pricing/) subscription is required for full access.

- **Duration:** 2.1 hrs
- **Level:** Intermediate
- **Access:** [Scrimba Pro](/docs/pricing/) required

## What Makes This Course Distinctive

Astro is taught as a way to ship content-heavy pages with component reuse and Markdown in the loop, which matches how blogs, docs, and marketing sites really get built. The course walks you from project setup through deployment of a personal site, so file structure and build output stay grounded in something you might publish. Interactive screencasts help because Astro's island architecture and config can feel abstract until you edit a real layout. You get a pragmatic taste of performance-minded defaults without turning the lesson into a benchmark contest. The tone fits developers who already write HTML and JS and want a modern static-first workflow. It is not the place for deep React SPA state management or a full CMS architecture review.

## What You'll Build

- Project setup and overview

These hands-on projects reinforce what you learn and give you portfolio-ready work to show employers.

## Prerequisites

Basic knowledge of HTML and CSS is recommended before starting this course.

## Who Is This Course For?

Developers who maintain personal sites, documentation, or landing pages and want components plus Markdown without shipping a giant client bundle by default. Also useful if you outgrew plain HTML/CSS but do not need a full client-rendered app.

**Not ideal if:** Your goal is a complex authenticated dashboard with heavy client state, or you only want to deepen React skills and ignore static site workflows.

## Choose This If

**Choose this course if:**

- You want to build and deploy a real Astro project while typing along in the browser.
- You care about fast content sites and want to see how Astro structures pages and components.
- You are comparing Astro to other meta-frameworks and want a short, structured first pass.
