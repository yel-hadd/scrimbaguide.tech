## Why TypeScript at all

TypeScript is JavaScript with static types. It catches a class of bugs at compile time instead of runtime: typos, wrong argument types, missing properties on objects. In 2026 it is the default for new React and Next.js projects, and the [official TypeScript handbook](https://www.typescriptlang.org/docs/handbook/typescript-from-scratch.html) puts the prerequisite plainly: you cannot learn TypeScript without first knowing JavaScript.

## What you will learn

- Type inference: how TypeScript infers types when you do not annotate
- Union types: `string | number` for values that can be either
- Interfaces vs types: defining object shapes and reusable types
- Generics: `<T>` for reusable, type-safe functions and components
- Utility types: `Partial`, `Pick`, `Omit`, and others

## TypeScript with React

The course covers typing React components: props interfaces, generic components, and event handlers. For example:

```tsx
interface ButtonProps {
  label: string;
  onClick: () => void;
}

function Button({ label, onClick }: ButtonProps) {
  return <button onClick={onClick}>{label}</button>;
}
```

## TypeScript with Express

You will type Express routes, request and response objects, and middleware, which matters for building backends where the request shape is enforced before logic runs.
