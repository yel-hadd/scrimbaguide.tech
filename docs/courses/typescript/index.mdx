## Why TypeScript?

TypeScript is JavaScript with static types. It catches bugs at compile time instead of runtime — typos, wrong argument types, missing properties. In 2026, TypeScript is the default for new React and Next.js projects. The [official handbook](https://www.typescriptlang.org/docs/handbook/typescript-from-scratch.html) puts it simply: you can't learn TypeScript without learning JavaScript, but TypeScript makes your JavaScript safer and easier to maintain.

## What You'll Learn

- **Type inference** — TypeScript infers types when you don't annotate
- **Union types** — `string | number` for values that can be either
- **Interfaces vs types** — defining object shapes and reusable types
- **Generics** — `<T>` for reusable, type-safe functions and components
- **Utility types** — `Partial`, `Pick`, `Omit`, and more

## TypeScript + React

The course covers typing React components: props interfaces, generic components, and event handlers. For example:

```tsx
interface ButtonProps {
  label: string;
  onClick: () => void;
}

function Button({ label, onClick }: ButtonProps) {
  return <button onClick={onClick}>{label}</button>;
}
```

## TypeScript + Express

You'll learn how to type Express routes, request/response objects, and middleware. Critical for building type-safe backends.
