# How to Learn JavaScript in 2026

<DisclosureNotice />

**Quick Answer:** Learn **syntax + problem solving** first, then **async + modules**, then **DOM or Node**, then **one framework (usually React)** while shipping **3–5 real projects**. Expect **4–9 months** part-time for hireable basics—speed depends on hours/week, not talent.

<!-- truncate -->

## Data points to calibrate your roadmap

- **JavaScript is still the default web language:** Recent Stack Overflow survey data continues to rank JavaScript as the most used language among professional developers. Source: [Stack Overflow Developer Survey](https://survey.stackoverflow.co/2025/technology#most-popular-technologies-language-prof).
- **Developer jobs are not disappearing:** U.S. labor projections show strong long-term demand in software and web roles, with web developer and digital designer roles projected at **16% growth** from 2023 to 2033. Source: [BLS Web Developers and Digital Designers](https://www.bls.gov/ooh/computer-and-information-technology/web-developers.htm).
- **Earnings potential is real, but skill depth matters:** the U.S. median pay for software developers remains above six figures; employers still filter heavily for portfolio quality and practical problem solving. Source: [BLS Software Developers](https://www.bls.gov/ooh/computer-and-information-technology/software-developers.htm).

## Phase 1 — JavaScript the language (4–8 weeks part-time)

- Variables, functions, arrays, objects, loops
- `map` / `filter` / `reduce`
- Errors, debugging in DevTools
- **Project:** CLI quiz, expense tracker, or small API consumer

Use MDN plus an interactive course. Scrimba’s [Learn JavaScript](/docs/courses/javascript/) is built for typing inside lessons; supplement with reading when concepts feel fuzzy.

## Phase 2 — Async & modern modules (2–4 weeks)

- Promises, `async/await`
- `fetch` and basic HTTP
- ES modules and bundler basics (Vite is fine)

**Project:** Weather dashboard or GitHub user lookup that handles loading + errors.

## Phase 3 — Browser or Node (pick one spine)

- **Browser track:** DOM manipulation, events, accessibility basics
- **Node track:** Express/Fastify hello world, environment variables

Most job-bound beginners should bias **browser + React** unless you already know you want backend-only.
